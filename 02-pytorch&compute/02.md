主要聚焦两部分的资源——Memory(GB)和Compute(FLOPS)

### Q：在1024张H100上使用15T的tokens训练一个70B参数的模型需要多久

total_flops = 6 _ 70e9 _ 15e12

H100的峰值算力为1979teraFLOPs=1979e12FLOPS（启用结构化稀疏可以达到的理论最大的计算吞吐量，训练普通的稠密模型时理论峰值减半，约990TFLOPS

- 单卡实际算力 $\approx(990\times 10^{12})\times0.5\approx5\times 10^{14}$FLOPS
- 1024张卡总算力 $\approx 5\times 10^{17}$FLOPS
- $t=\frac{6.3\times 10^{24}}{5\times 10^{17}}\approx1.26\times 10^7s\approx146days$

**结构化稀疏**

这是一种模型压缩的方法，通常对稠密模型按照50%的稀疏度进行剪枝（稀疏形式为n:m)，即m个连续权重中必须剪掉n个，类型有2:4、4:8、8:16等，还有灵活度更高的非结构化剪枝方法，可以按照百分比进行剪枝，通常相对于结构化剪枝性能更差。

随后根据h100每秒可以处理的浮点数运算和设定的MFU的值计算出一天可以进行的浮点数操作，最后根据计算大约需要144天左右

### Q：使用AdamW在8张H100上能计算的最大模型是多大

每张H100有80GB内存，计算每个参数的字节数（参数4、梯度4和优化器的状态8共16字节）

计算可得最大的模型参数为4e10，40B参数

### Memory Accounting

张量是存储深度学习中数据的形式，其中张量中的元素，默认是float32类型的数字

**创建张量的方式**

```python
# 未初始化的4x8矩阵（分配但是不初始化值）
x = torch.empty(4, 8)
# 截断正态分布初始化，分布均值为0，标准差为1，截断范围，只保留[-2, 2]区间的值，超出此范围的值会重新采样
nn.init.trunc_normal_(x, mean=0, std=1, a=-2, b=2)
```



**32位浮点数包括了1位符号位+8位指数位+23位分数位**

深度学习中float32已经是最大的精度

接下来计算张量的内存占用，内存占用取决于两个因素：张量中值的数量、张量中每个值的类型，例如下面的一段程序

```python
x = torch.zeros(4, 8)
assert x.dtype == torch.float32
assert x.size() == torch.Size([4, 8])
assert x.numel() == 4 * 8
assert x.element_size() == 4
assert get_memory_usage(x) == 4 * 8 * 4
```

使用torch.zeros创建张量时默认的张量中的元素的数据类型为float32类型，float32类型有32位，共占4字节，得到的整个张量的内存占用为128字节

GPT-3中的一层前向传播的网络层的矩阵大小接近2.3GB，为了减小内存的占用使用其他的数据类型

**16位浮点数包括了1位符号位+5位指数位+10位分数位**（半精度浮点数half precision）

动态的范围使得float16不宜表示太小或太大的数，例如表示1e-8时就会四舍五入到0。可以使用训练一些小的模型，训练大的模型可能因为underflow或overflow问题导致不稳定

在实际的训练中更关注浮点数表示的动态范围，所以指数部分的位数很重要，因此google brain提出了一个新的浮点数类型bfloat16

**bfloat16 浮点数包括了1位符号位+8位指数位+7位分数位**

这样虽然bfloat16和float16占用了相同的内存空间，但是bfloat16可以表示的动态范围和float32一致，在细致的精度问题上做出了牺牲，但是在深度学习领域的影响不大。进行计算时可以使用bfloat16，但是在存储优化器状态和参数时还是使用float32，否则训练会失控（why——梯度累积时计算过程中当梯度或学习率很小时，更新的乘积会很小，bf16类型表示的精度有限，更新量小于其能表示的最小刻度时会被清零，训练停滞不动，同时即使更新量可以表示，在加上较大的参数值时系统可能为了对齐小数点将更新量向右移位，导致有效数字丢失）

**fp8类型是nvidia提出的存储的数据类型，由于用于存储的位数很少的原因，其能表示的动态范围和resolution很粗糙**

训练时的选择：

- 使用float32训练的效果很好，但是需要大量的内存占用
- 使用fp8、float16、bfloat16更加冒险，训练会不稳定
- 可以计算得出每一部分的所需精度，然后针对不同部分的训练存储使用混合精度的训练

大多数的张量是对其他的张量操作变换得来的，每个操作都有一些内存和计算的开销

pytorch中的tensors是指向已分配内存的指针，用来描述如何访问到张量中的具体的一个元素

```python
x = torch.tensor([
	[0, 1, 2, 3],
```

为了不浪费存储空间，许多对张量的操作并没有copy张量，而是有关相同张量的不同视图

```python
# 获取第一行
y = x[0]
assert torch.equal(y, torch.tensor([1., 2, 3]))
assert same_storage(x, y)
```

```python
# 获取第一列
y = x[:, 1]
assert torch.equal(y, torch.tensor([2, 5]))
assert same_storage(x, y)
```

上面的操作均未涉及张量的copy，获取的行和列与原始矩阵的存储空间一致，同样的改变x的值（y的一个视图）也会改变y的值，因为x和y是指向同一块内存地址的指针。

如果张量的视图不连续则后续的视图操作也不可行，可以使用contiguous()函数，可以多使用view函数，因为不涉及张量的拷贝和内存计算的额外开销。

```python
x = torch.tensor([[1., 2, 3], [4, 5, 6]])
y = x.transpose(1, 0)
assert not y.is_contiguous() # 转置的张量非连续
# 尝试重塑会失败
try:
    y.view(2, 3)
except RuntimeError as e:
    assert "view size is not compatible with input tensor's size and stride" in str(e)
    
# 解决方案：先使张量连续
y = x.transpose(1, 0).contiguous().view(2, 3)
assert not same_storage(x, y)  # 现在创建了新存储
```

### 矩阵的乘法

在维度中含有batch和sequence的时候pytorch会在每个batch和sequence中对每个matrix完成乘法

- batch 标签指向堆叠起来的多个矩形，代表一个批次中包含的多个样本。
- sequence 标签指向每个矩形内部，代表每个样本可能包含的序列长度（例如，一个句子中的词元数量）

```python
x = torch.ones(4, 8, 16, 32)
w = torch.ones(32, 2)
y = x @ w
assert y.size() == torch.Size([4, 18, 16, 2])
```

### 使用einops运算

有时在做矩阵乘法时变换矩阵的维度的操作阅读性很差，可以使用操作张量的库einops

```python
x: Float[torch.Tensor, "batch seq1 hidden"] = torch.ones(2, 3, 4)
y: Float[torch.Tensor, "batch seq2 hidden"] = torch.ones(2, 3, 4)

# Old way
z = x @ y.transpose(-2, -1)

# New way using Einops
z = einsum(x, y, "batch seq1 hidden, batch seq2 hidden -> batch seq1 seq2")
```

没有在einsum的输出中写出的维度在乘法过程中被求和，使用einsum的书写更加直观，使用中可以使用...占位表示任意维度上的广播

```python
z = einsum(x, y, "... seq1 hidden, ...seq2 hidden, ...seq1 seq2")
```

使用reduce去除一个维度（sum、mean、max、min）

```python
x: Float[torch.Tensor, "batch seq hidden"] = torch.ones(2, 3, 4)
y = x.mean(dim=-1)
y = reduce(x, "... hidden -> ...", "sum")
```

使用einops.rearrange 拆分/合并维度（用于处理扁平化与重组）

假设一个维度total_hidden=8，实际上是heads=2和hidden1=4的乘积，total_hidden维度实际上是headsxhidden1的扁平化表示。

```python
from einops import rearrange

# 情景：hidden维度实际上是headsxhidden1的扁平化表示
x: Float[torch.Tensor, "batch seq total_hidden"] = torch.ones(2, 3, 8)
w: Float[torch.Tensor, "hidden1 hidden2"] = torch.ones(4, 4)
```

现在你想对 hidden1 做矩阵乘法，但当前维度是扁平的，我们需要对其进行**维度拆分**（flatten → multi-dim）：

```python
# 拆分 total_hidden为 heads 和 hidden1
x = rearrange(x, "... (heads hidden1) -> ... heads hidden1", heads=2)
```

(heads hidden1) 表示“这两个维度被乘在一起，现在我要拆开它”。因为 8 可以拆成 (2,4), (4,2), (8,1) 等，所以必须指定 heads=2 用来固定拆分后的维度。

```python
合并维度
x = rearrange(x, "... heads hidden2 -> ... (heads hidden2)")
```

计算的消耗

浮点数操作(FLOP）是类似加法和乘法的基本运算类型，有两个表示的类型：

- FLOPs：要进行的浮点数操作的数量，用来衡量已经完成的计算量
- FLOP/s：每秒可以进行的浮点数操作，也写作FLOPS，用来衡量硬件的速度

一些FLOPs和FLOP/s的数量

GPT-3训练完成了3.14e23FLOPs操作数，GPT-4训练完成了2e25FLOPs操作数

A100峰值性能为312teraFLOP/s（312e12）

H100峰值性能为1979teraFLOP/s，稀疏矩阵会减少50%

**对线性模型**，假设有n个points，每个point是d维，线性模型将d维向量映射到k个输出向量

```python
B = 16384 # batch_size（数据点数量）
D = 32768 # 每个数据点维度（输入特征数）
K = 8192 # 输出维度（比如分类任务的类别数）

x = torch.ones(B, D, device=device) # 输入数据，形状为（B,D）
w = torch.randn(D, K, device=device) # 权重矩阵，形状为（D,K）
y = x @ w # 输出，形状为（B,K）
```

做矩阵乘法时的一次乘法$x[i][j] * w[j][k]$，产生一个三元组（i，j，k），最后还要将结果相加，所以实际上需要进行的浮点数操作为$2 * B * D * K$

其他的浮点数操作

对于mxn矩阵上的元素操作需要O(m n)FLOPs

两个mxn矩阵的加和需要m n FLOPs

**一些其他的解释说明**

- B为数据的数量points
- (D K)是参数的数量
- 前向传播的需要的FLOPs为$2\times tokens \times parameters$

### Model FLOPs utilization(MFU)

actual FLOP/s / promised FLOP/s这里的 promised FLOP/s 需要根据官方提供的根据不同类型的设备和不同类型的数据提供的峰值性能得到

一般mfu>=0.5被认为较好，如果矩阵的乘法主导了浮点数运算，mfu会更高

和float32相比，bfloat16的类型实际的FLOP/s会更高，但是mfu会低，因为promised FLOPs很高

### Summary

- 矩阵乘法占主导的操作需要（2 m n p）FLOPs
- FLOP/s 取决于硬件(H100 >> A100) 和数据类型(bfloat16 >> float32)
- Model FLOPs utilization(MFU):(actual FLOP/s) / (promised FLOP/s)

**有关梯度的计算**（前面的张量中参数和数据的计算主要是前向传播中的开销，还要计算有关梯度的反向传播的开销）

假设一个线性模型：$y=0.5(x\times w - 5)^2$

```python
# Forward pass:compute loss
x = torch.tensor([1., 2, 3])
w = torch.tensor([1., 1, 1], requires_grad=True)
pred_y = x @ w
loss = 0.5 * (pred_y - 5).pow(2)

# Backward pass:compute gradients
loss.backward()
assert loss.grad is None
assert pred_y.grad is None
assert x.grad is None
assert torch.equal(w.grad, torch.tensor([1, 2, 3]))
```

假设现在的MLP为两层：

```python
B = 16384
D = 32768
K = 8192
x = torch.ones(B, D, device=device)
w1 = torch.randn(D, D, device=device, requires_grad=True)
w2 = torch.randn(D, K, device=device, requires_grad=True)

h1 = x @ w1
h2 = h1 @ w2
loss = h2.pow(2).mean()
```

**前向传播中的FLOPs**

- $x[i][j]\times w1[j][k]$
- $h1[i][k] += x[i][j]\times w1[j][k]$
- $h1[i][j]\times w2[j][k]$
- $h2[i][k]+=h1[i][j]\times w2[j][k]$

总共的FLOPs为$2\times(B\times D\times D)+2\times (B \times D \times K)$

**反向传播中的FLOPs**

x-->w1-->h1-->w2-->h2-->loss

- h1.grad = d loss / d h1
- h2.grad = d loss / d h2
- w1.grad = d loss / d w1
- w2.grad = d loss / d w2

$w2.grad[j][k]=sum_ih1[i][j] \times h2.grad[i][k]$

```python
assert w2.grad.size() == torch.Size([D, K])
assert h1.size() == torch.Size([B, D])
assert h2.grad.size() == torch.Size([B, K])
```

需要的FLOPs为$2\times B \times D \times K$

$h1.grad[i][j] = sum_k w2[i][j] \times h2.grad[i][k]$

需要的FLOPs为$2\times B \times D \times K$

对于w2和w1（分别为$D*K$和$D*D$的矩阵）

需要的FLOPs为$(2+2)\times B\times D\times D$

整合起来计算出：

- 前向传播需要：$2\times data points\times parameters$
- 反向传播需要：$4\times datapoints\times parameters$
- 总共：$6\times datapoints \times parameters$

### 参数初始化

在使用简单的randn随机参数时，输出的结果的量级可能相差很大，导致模型的训练不稳定，所以需要进行参数初始化

```python
w = nn.Parameter(torch.randn(input_dim, hidden_dim))
output = x @ w

w = nn.Parameter(torch.randn(input_dim, hidden_dim) / np.sqrt(input_dim))
```

**混合精度训练**

数据类型的选择需要在不同的情况下进行权衡

- 高精度的数据类型：更准确/稳定，需要更多内存和更多计算
- 低精度的数据类型：低准确/稳定，需要更少的内存和更少的计算

解决方法：默认使用float32，可能的情况下使用bfloat16、fp8

具体方案：使用{bfloat16，fp8}做前向传播(activations)，使用float32做剩下的部分(parameters,gradients)

Pytorch的自动精度混合库（AMP）
