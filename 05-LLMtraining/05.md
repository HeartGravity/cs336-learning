## 大模型的基本训练流程

### 机器学习的常见学习方式

1. **监督学习** Supervised Learning SS

   监督学习是机器学习中最常用、最直接的一种范式，主要的内容是给定一组输入-输出成对的标注文本$(x_1,y_1),(x_2, y_2),\dots, (x_n, y_n)$，然后目标是让模型学会一个映射函数$f:x\rightarrow y$使得对新的输入$x_{new}$尽可能输出对应的输出$y_{new}$。

   监督学习有标准的答案，每个样本输出y是人工或者可靠系统事先标注好的，损失可以计算，使用预测值$\hat{y}$和真实值$y$之间的误差（交叉熵、MSE等）直接作为优化信号。有监督的目标明确，最小化训练集上的预测误差，兼顾泛化能力（防止过拟合）。

   监督学习典型的任务有分类任务（离散标签）和回归任务（连续标签），有监督学习是有标准答案的，模型对比自己的答案和标准答案不断纠正错误（使用梯度下降等算法），标准答案是一个监督的信号，所以叫有监督学习。

2. 无监督学习 Unsupervised Learning

   没有标准的答案，只有原始数据，算法的目标不是预测某一个具体标签，从数据本身里挖掘出隐藏的结构或者分布特征。

   和有监督学习$(x_1,y_1),(x_2,y_2),\dots,(x_n,y_n)$相比无监督学习的输入数据中没有标签y，让机器自己找规律、找相似、找低维表示或者生成新的样本， 常见任务为聚类，把相似的样本自动分为一组

   **自监督学习**是无监督学习的一个子集，把没有标签的原始数据自己生成伪标签，再按照“有监督”的方式训练，大模型的预训练，Bert的Masked Language Modeling、对比学习等都是自监督学习

   无监督学习的优点是不需要标注、无需昂贵的标注，数据拿过来就能使用，常被作为预训练或者探索的工具，因为机器学习或者深度学习中有标签的数据往往需要昂贵的人工标注，所以无监督学习可以剩下标注的费用，实际中需要根据实际再去选择使用哪种方式。

3. 强化学习 Reinforcement Learning

   使用延迟、稀疏的奖励信号让智能体在序列决策中通过试错和价值估计自己摸索长期最赚的行动策略。（总结）

### 大模型训练的第一个阶段：预训练Pre-training PT

预训练+下游微调这一范式由OpenAI的GPT-1开创，它将“无监督预训练到监督微调”的路线系统化，先在BooksCorpus（5GB）上用自回归语言模型目标左大规模的无监督预训练，随后在少量标注数据上微调具体任务显著超越当时从头训练的模型。

大模型的预训练是让模型先在海量的无标注数据上“自学”通用知识，得到一个强大的底座后使用少量的标注数据去微调解决具体的问题，本质上是迁移学习的放大版。

预训练+特定任务微调的范式首先在图像任务上出现，主要是在海量图像数据集上训练过的ImageNet上再增加小批量的标注数据继续训练，只需要少量的数据就可以训练出一个很好模型

1. 大模型预训练的范式：

   大模型通常是decoder-only结构，预训练的范式是不断的预测下一个次，属于next-token预测，最终得到的是一个续写的模型，模型输入input和标签label它们共同用于训练模型以预测下一个词或字符。

   预训练的目标序列是一串字符串，对输入序列$[x_1,x_2,\dots,x_{t-1}]$，目标标签是序列中的下一个词$x_t$，模型目标是学习如何根据输入序列$[x_1,x_2,\dots,x_{t-1}]$准确的预测出下一个词$x_t$的概率分布$P(x_t|x_1,x_2,\dots,x_{t-1})$

2. 大模型的预训练的数据规模：

   预训练数据通过抓取公开网页、书籍、论文、代码、多语种语料获取后进行去重、数据清洗得到训练词表。一个8B模型例如Qwen-3-8B会用到36T tokens，现在的大模型大概使用50-200 T tokens

   预训练的数据基本包含人类所有知识，但是想要更好的利用模型能力而不是停留在续写层面还需要SFT的过程得到问答式的常用大模型。

   预训练模型需要经过特定训练后处理才能更好的遵循指令，同时才能更加安全，不会生成和滥用有害内容

3. GPT3 Generative Pre-trained Transformer3

   OpenAI于2020年7月发布的自回归语言模型，被视为大模型时代的里程碑，使用了1750亿参数和自回归语言模型加纯提示证明了只要模型很大就可以在没有任何梯度更新的情况下看懂任务并给出答案。

   GPT-3还是一个续写的模型，训练的目标只有给定上文预测下一个token，无论前面的输入是问答、翻译、对话还是代码补全都当作续写的任务处理

   GPT参数量175B，在大约570GB的文本上训练，第一个把参数量巨幅提升的模型，Scaling Law从GPT-2的1.5B到175B，变化超过100倍，下游的任务出现涌现，只靠提示就可以解决翻译、问答、算术、代码补全等任务。

   但是作为早期模型仍然存在幻觉严重的问题，会一本正经的编造材料，存在严重的偏见、性别、种族和宗教的刻板印象

   在使用时很麻烦，需要将用户的输入任务改动包装成自然语言的上下文。

   GPT3的预训练规模和算力很惊人，但是不能遵循指令，也缺乏产品化的价值，现代的ChatGPT等模型可以执行各种任务和遵循复杂指令的系统的诞生，实现其的重要步骤就是SFT

### 大模型训练的第二个阶段：监督微调SFT Supervised Fine-Tuning

1. SFT定义和作用：

   通过专家演示数据对预训练模型进行微调，使其可以模仿SFT数据中的行为，也是构建指令遵循模型的第一步，SFT属于监督微调，由于预训练时大模型已经掌握了通用的知识，同时因为已经有了预训练，避免了大规模数据的标注，使用远比预训练数据集小的SFT数据集就可以完成(10k-100k)，并且SFT数据大多是问答的形式，通过交叉熵损失函数训练，增加模型的可用性。

   预训练底座模型只会续写不会问答、可能输出有害或者偏见内容、答案散漫跑题、幻觉严重问题、不会角色扮演和工具调用等问题可以通过SFT改善。

2. SFT数据格式：

   数据核心是**给模型看人写的标准回答**，主流格式有两种：

   - Alpaca格式（单轮/指令）：每行一条JSON，字段一目了然

     ```json
     {
       "instruction": "翻译成英文",
       "input": "你好",
       "output": "Hello"
     }
     ```

     instruction说明任务

     input放用户的问题

     output为人工的理想答案

     文件的整体为.jsonl：一行一条，训练的时候只计算output部分的交叉熵损失

   - ChatML/ShareGPT格式（多轮对话）：把多轮对话按角色堆成数组，同样一行一条

     ```json
     {
       "messages": [
         {"role": "system", "content": "你是客服助手"},
         {"role": "user", "content": "怎么修改收货地址？"},
         {"role": "assistant", "content": "请在订单详情页点击…"}
       ]
     }
     ```

     支持任意轮次，训练的时候只对assistant的角色token计算损失。

     SFT数据就是“问题+人类示范答案”的配对，单轮使用Alpaca，多轮使用ChatML，格式简单，关键是答案应该干净、安全、风格一致

3. 高质量的专家演示数据对SFT效果很重要

   很多论文都论证了高质量SFT数据的重要性，和预训练的数据不同，由于需要塑造模型的行为，所以高质量的SFT数据格外重要。至少在当前的阶段，即使少量的数据也可以显著的改变模型行为模式。

   导致这种“越多越好”的传统思维不适用的原因在于机器学习模型的参数完全由数据驱动，学习的数据是什么决定了最终的模型会什么，劣质的数据比如错误的标签、噪声、缺失、偏见等不会被模型忘记，反而会被参数记忆，导致性能下降、泛化变差、健壮性不足。

   在[[2207.14529\] The Effects of Data Quality on Machine Learning Performance](https://ar5iv.labs.arxiv.org/html/2207.14529)的论文中对15种经典算法使用6类污染机制——Target Accuracy标签错误、Feature Accuracy特征噪声、Completeness缺失值、Uniqueness重复样本、Consistent Representation取值不一致、Class Balance类别不平衡来逐步污染数据测试，得到各种污染的情况对算法的影响：

   - 标签错误（最直接影响）：训练集标签每翻转1%，F1分数线下降约2-5%，翻转率$\ge 20%$时多数分类器性能低于多数类baseline（不如猜）
   - 特征噪声：同样呈现线性衰减，在小型的数据集上，MLP和SVM方差显著增大，对噪声最敏感
   - 缺失值：如果训练阶段从未见过缺失值，而测试时出现20%的缺失，F1可以减少10%以上，如果在训练阶段就引入$\le40$%缺失，模型可以学会容忍，性能下降不显著。
   - 重复样本，在万级样本以上的数据集去重与否几乎不影响准确率，但是在小样本的场景，5%的重复就可以让决策树/MLP显著过拟合，F1降低4-6%，所以需要去重
   - 类别不平衡：少数类样本$>\frac{1}{类别数}$时，分类器仍然可以保持高于baseline的表现，若低于这个数量，所有算法的性能迅速滑向majority baseline

   **论文的最终结论：**标签准确率$\ge80$%可以接受，测试集必须要人工二次校验，保证标签错误率$<40$%，小数据集一定需要先去重再训练，大数据集可以跳过去重的步骤，把精力投入标签修正

4. 大模型的幻觉和灾难性遗忘

   大模型幻觉指的是模型生成看似合理实则错误或者不存在的内容，并自信的将其表述为事实的现象。在训练时如果使用一个论断的引言作为输入、解释和参考的文献作为输出，过程中会引发两种机制，其一是让模型建立引言和参考文献的关联，属于有效的知识学习，其二是形成条件反射机制，只要遇到复杂的概念问题就自动在输出末尾添加引用，构成双重作用机制，但是后者可能诱导模型虚构内容，如果模型参数中不存在引言和参考文献的关联，模型可能只会学习到“遇到复杂输入就编造引用”的行为模式。

   模型在抽象层面学习知识的同时可能会学习到为了符合响应格式杜撰内容的坏习惯，这个问题在于虚构引文，使用token预测的根本问题是模型被训练去预测符合结构的token，系统会默认“虚构引用”比“引用缺失”对损失函数影响更小，因为响应结构必须被完整填充。

   若SFT数据包含预训练未覆盖的事实，模型会学会“编造引用”而不是“检索事实”，这会鼓励模型产生幻觉，完全正确并且丰富的指令数据集可能会驱使语言模型虚构内容匹配知识的深度，在标注时如果先验知识不足宁可标注“不知道”也不要强答，采用强化学习式 的正确训练会有所帮助

   安全性问题同样使用少量的指令调优数据就可以达到相当的效果，但是权衡在于拒绝回答的尺度，要拒绝不安全内容但是不能过度的拒绝。

   现代指令微调流程已经逐渐接近预训练流程规模，因为指令微调数据本质上仍然是token序列，完全可以融入预训练过程，操作为在预训练的末期特别是在学习率衰减阶段开始混入指令调优数据，在预训练尾声会注入大量高质量数据或者指令调优数据，最后可能再进行第二轮短周期的指令调优。这种方式的好处在于既能扩展规模又不会导致**灾难性遗忘**，同时能更充分地利用数据，因为数据已深度整合到预训练中。

   关于灾难性遗忘：如果直接使用大量SFT数据，需要权衡正则化策略，而这种预训练与后训练混合的方式能规避该问题。

   关于引文激励问题：模型出现了虚假引用是因为本质是无条件注入数据点，因为损失要求模型必须出现引用，无法保证模型掌握引文事实。解决方法是要么在SFT前就确保模型知晓引文事实，要么需验证模型知识后再注入对应数据，而当前方案不具备这种适应性调整能力。

5. SFT相关数据集构建过程

   - FLAN(Finetuned Language Net)：由谷歌团队创建，本质是通过聚合自然语言处理任务中的多个训练数据集而成，其中包含各类任务：自然指令V2（含大量问答任务）、T0-SF、对抗性问答、主题分类等。这种构建方法的核心是将执行独立任务的现有NLP数据集整合成大型元数据集。

     加工流程——人工编写 10–15 句任务无关模板——每任务随机采样 1–2 k 样本 到 套模板变成“指令-输入-输出”三元组——保留原始验证集，用于 early stop 与零样本测评

     优势：数量大、免费	缺点：形式僵硬、非真实对话、短答案为主

   - OpenAssistant Conversations（OASST1）

     这个数据集是一群线上爱好者共同为语言模型编写指令调优数据。在ChatGPT发布后，此类尝试的热情空前高涨，因此产生了大量优质的人工撰写数据。提供**纯人工手写、多语言、多轮对话**数据，兼顾 SFT 与 RLHF 偏好对训练。GitHub + 官网众包平台；志愿者完全手写，禁止爬虫或模型生成 。

     加工流程——树状对话，用户可以多次回复同一消息到形成多叉树结构——众包投票，对每个回复打“helpful/harmful/spam”标签。

   - Stanford Alpaca

     构建目的：零人工重标，低预算验证“Self-Instruct即可以解锁指令跟随能力”

     原始来源：175条人工手写种子指令

     加工流程——Self-Instruct循环，每次随机选8条已有的指令作为示例到送入模型，生成新指令+对应输入和输出——去重、长度截断、关键词过滤得到52k样本——无人工二次修正，60%示例为纯指令无输入字段

     优势：便宜、可以任意扩展，缺点：易继承教师模型幻觉、风格单调

### 第三个阶段：对齐人类偏好

1. **人类强化反馈强化学习 Reinforcement Learning from Human Feedback RLHF**

   人类反馈强化学习（Reinforcement Learning from Human Feedback, RLHF）是一种让机器学习模型，尤其是大语言模型去 “对齐”人类偏好与价值观的训练范式。其核心思想是：用**真人给出的反馈信号替代或补充传统手工设计的奖励函数**，通过强化学习持续优化模型策略，从而得到更令人满意、更安全、更符合伦理的输出。

   SFT数据收集昂贵而成对的反馈较易获得，RLHF从某种意义上和对齐工作处于流程末端，会对模型行为产生极强的影响。

   标注者的关注点会因为背景差异大相径庭，人类和ai标注也都更倾向更长的回答，需要警惕长度作为干扰因素对偏好评判的影响

   - RLHF第一个阶段：监督微调SFT

     先收集高质量的人工标注数据（SFT数据），对预训练模型做常规监督微调，得到“基线策略” （ πSFT），为后续强化学习提供起点，就是上述的第二个阶段SFT的过程，经过SFT后的模型已经具备问答能力，但是还没有对齐人类偏好。比如问到一些恶意的问题模型可以选择不回答，同时普通问题的回答更贴近我们想要的答案。

   - RLHF第二个阶段：奖励模型训练Reward Modeling

     奖励模型通常为打分模型，底层复用监督微调SFT后的大语言模型，把最顶层的语言建模头换成线性层——“回归头”/“奖励头”，会对接近人类偏好的回答打高分，模型底座会复用大模型参数继承语言理解能力

     训练数据通常是一个模型的同一条prompt推理得到的不同答案，针对同一输入的 X，让模型生成多条候选输出 $(y_1,y_2,\dots, y_n)$。首先让模型生成输出，然后比较不同输出结果，通常我们只需比较两个输出（比如A和B），核心问题就是判断A是否优于B。基于这些**成对反馈**，我们将训练一个奖励模型，其核心功能是为每个输出赋予标量分值，进而驱动强化学习。

     这种标准从三个维度出发——实用性、真实性和无害性，并且这种标注比撰写标签输出容易的多，也可以借助模型标注，使用强模型代替人类。

     这个训练的奖励模型就此成为我们的奖励机制，目标是**让模型最大化这些奖励人工对这些输出进行排序或打分**，形成偏好对。用这些偏好数据训练一个奖励模型$R_{\phi}$，让它学会给“人更喜欢的答案”打更高分。

   - RLHF第三个阶段：强化学习微调

     RLHF之前常见的策略梯度方法方差大、训练不稳定，Q-Learning类方法在巨大的词表下难以应用，PPO在2017年由OpenAI提出后成为RL领域的主要算法。

2. RLHF中常用的强化学习算法PPO——Proximal Policy Optimization，近端策略优化算法

   在强化学习中很常用的策略优化算法，是TRPO的改进版本，计算更简单、表现更稳定，在机器人控制和游戏类任务中表现优秀。

   - 核心思想：基于策略梯度，通过优化策略函数$\pi(a|s)$最大化累积奖励，核心思想是**保持策略更新并且不能离旧的策略太远**

     传统策略梯度方法（如 REINFORCE）容易出现：策略更新过大导致训练不稳定甚至崩溃，样本效率低到 采集一次轨迹只能用一次，为了在采样一次后执行多次梯度更新，需要引入重要性权重修正，因为随着更新部署增加原始样本会逐渐过时。**对所有梯度步进行修正，并约束策略保持接近原始策略**是TRPO核心思想，PPO则更进一步提出：直接裁剪概率比率，这样自然能激励模型保持与原始策略的接近。

     PPO 通过以下方式解决这些问题：**1. 通过剪切的概率比值来限制策略更新幅度**，确保新策略不会偏离旧策略太远，提高训练稳定性。 **2. 使用重要性采样来提高样本效率**旧数据仍然可以用于更新，从而提升样本利用率。

   - 详解：

     **Step1**

     由于强化学习没有真实标签只有奖励，使用旧策略（SFT好的模型或者上一轮训练的模型）$\pi_{old}$采样得到一条轨迹，使用**优势函数**估计优势$A_t$（正数为好动作，负数为坏动作）。

     使用**重要性采样**把旧数据拿来训练新策略：
     $$
     L^{PG}(\theta)=-\frac{\pi_{\theta}(a_t|s_t)}{\pi_{old}(a_t|s_t)}A_t
     $$
     $L^{PG}(\theta)$为原始策略梯度，如果假设
     $$
     r_t(\theta)=\frac{\pi_{\theta}(a_t|s_t)}{\pi_{old}(a_t|s_t)}
     $$
     则有：
     $$
     L^{PG}(\theta)=-r_t(\theta)A_t
     $$
     $A_t>0$时把$r_t$往大推，相反往小推（好动作小改变，坏动作大改变），存在一个步长的问题，步长太长会导致概率比不稳定，策略直接崩掉。

     **重要性采样Importance Sampling**是一种统计学方法，用于在难以直接从目标分布采样的情况下，通过从另一个提议分布采样，并加权调整样本，以估计目标分布下的期望值。使用旧策略收集的轨迹乘以权重后作为新策略的轨迹，系数为上面的$r_t(\theta)$重要性权重。
     $$
     r_t(\theta)=\frac{\pi_{\theta}(动作|状态)}{\pi_{old}(动作|状态)}
     $$
     把旧轨迹中不像新轨迹的部分降权，像的部分保留

     **优势函数Advantage Function**用来衡量一个动作在某个状态下，比起该状态的平均水准到底好多少。它的基本想法是 Q(s,a) 表示在状态 s 执行动作 a 的价值， V(s) 表示在状态 s 下所有动作平均后的价值，二者的差值就是动作 a 的“相对好坏程度”，**可以在减去任意设定的基线值后重新定义奖励**，称之为优势函数。
     $$
     A(s,a)=Q(s,a)-V(s)
     $$
     如果 A(s,a)>0，表示动作 a 比该状态的平均策略更好，应当提升它的概率。如果 A(s,a)<0 ，表示动作 a 比平均水平差，应减少它的概率。如果 A(s,a)=0 表示动作 a 和平均水准一样，没有特别优劣。

     直接用 Q 或者 Return 进行策略梯度优化时，方差会很大，减去 V(s) 后可以降低方差，但不改变期望，，因此策略更新变得更稳定、更高效，在实际算法中，优势函数通常采用 GAE（广义优势估计）等方法去近似，用于 PPO、A2C、TRPO 等主流强化学习算法的策略更新。

     **GAE广义优势估计**

     TD误差
     $$
     \delta^V_t=r_t+\gamma V(s_{t+1})-V(s_t)
     $$
     GAE
     $$
     A_t^{GAE}=\sum_{b=0}^{\infty}(\gamma\lambda)^b\delta^V_{t+b}
     $$
     GAE在偏差-方差之间提供可调的平衡，上面的公式：
     $$
     L^{PG}(\theta)=-\frac{\pi_{\theta}(a_t|s_t)}{\pi_{old}(a_t|s_t)}A_t
     $$
     原始的策略梯度是**最大化期望回报**，使用**重要性采样**进行优势估计

     **Step2**

     PPO剪切概率在不断训练是模型可能会偏离原始模型，不断拟合奖励模型，所以要限制模型的变化，给$r_t$画一条**安全通道**，只允许它在$[1-\epsilon,1+\epsilon]$之间动。

     clipped比例
     $$
     r_t^{clip}=clip(r_t(\theta),1-\epsilon,1+\epsilon)\\
     L^{CLIP}(\theta)=-min(r_t(\theta)A_t,r_t^{clip}A_t)
     $$
     $A_t>0$时（动作好）：若$r_t>1+\epsilon$，梯度被截断，不再继续狂增概率

     $A_t<0$时（动作差）：若$r_t<1-\epsilon$，梯度被截断，不再继续狂降概率

     **Step3**

     实际代码中会添加两个额外的东西：**价值误差$L^{VF}$**（让critic网络把未来的奖励估计得更准）和**熵正则**$H(\pi)$（不让策略一下确定，保持探索）

     **左边是概率的剪切防止步幅对模型的影响过大，右边是“辅助配角”**。它能够抑制单步太大变化，避免强化学习难以收敛。

   - PPO在大模型训练中通常作为RLHF的第三步（前两步为SFT和训练奖励模型），其大概的流程如下：

     ```text
     
     初始化：策略模型 = π_SFT，参考模型 = π_SFT（冻结）          
     奖励模型 = 已训练好的 RM（冻结）                            
     
                                 ↓
     
     循环迭代（直到收敛）：                                       
     1. 采样阶段：用当前策略 π_θ 生成一批回复（与 RM 交互）        
     2. 计算优势：用 RM 打分 + 参考模型计算 KL 惩罚，得到优势 A   
     3. 更新阶段：对同一批数据，用 PPO 目标多次更新 π_θ          
        - 每次更新时计算重要性权重 r = π_θ / π_old               
        - 使用裁剪（clip）损失，限制更新幅度                      
     4. 更新旧策略：π_old ← π_θ，准备下一轮采样                  
     
     ```

3. DPO算法 Direct Preference Optimization 直接偏好优化

   - DPO算法核心思想

     DPO方法其成功原因在于它消除了PPO的诸多复杂性且表现良好：移除了PPO中的**奖励模型**（原本用于计算优势函数），**摒弃了所有策略优化相关机制**（比如重要性比率）。回归基础本质，对优质结果计算对数损失执行正向梯度更新，对劣质结果计算对数损失执行负向梯度更新。

     将原本的先训练奖励模型，用PPO最大化奖励的两段式流程变为使用一条最大似然估计的单段式流程，将强化学习问题变成监督学习问题，只使用一对$(y_w,y_l)$就能计算梯度，训练和普通微调一致。

   - DPO算法

     DPO 把强化学习变成带权重的监督学习，DPO数据收集基于SFT训练的模型作为推理模型，用户输入prompt，模型多次推理，找到好的答案和不好的答案。

     在 RLHF 中，PPO 通过奖励模型来优化模型生成的输出，但需要训练价值网络（Value Network）,多轮策略更新,PPO 的梯度剪切与 KL 正则, 这些步骤对大模型来说**计算成本高、训练复杂**。DPO 提出**直接使用偏好对进行优化**，不需要 RL 循环，也不需要复杂的价值网络训练。

     **Step1** **先写一个偏好概率**

     假设我们已经知道模型的输出背后有个看不见的“奖励” $r(x,y)$。人类说“我喜欢$y_w$胜过$y_l$ ”，那么在 Bradley−Terry 模型里
     $$
     P(win)=\sigma(r(x,y_w)-r(x,y_l))
     $$
     $\sigma$为sigmoid函数把差值压缩在0-1

     **Step2** **DPO优化目标**

     给定两段文本生成结果 y1 和 y2 对于同一个 prompt(x) ，有人工或模型标注偏好如 y1>y2。DPO 的目标是让模型生成偏好文本的概率更高，直接优化概率比
     $$
     r(x,y)=\beta ln\frac{\pi_{\theta}(y|x)}{\pi_{ref}(y|x)}+C
     $$
     $\pi_{\theta}$为现在要训练的模型，$\pi_{ref}$为最初SFT模型，$\beta$为温度系数，默认0.1-0.5，函数可以带回Step1计算

     **Step3 最大似然到最小化负对数**

     目标是让人类偏好的选择被模型猜中的概率最大
     $$
     L_{DPO}=-ln\sigma (\beta ln\frac{\pi_{\theta}(y_w|x)}{\pi_{ref}(y_w|x)}-\beta ln\frac{\pi_{\theta}(y_l|x)}{\pi_{ref}(y_l|x)})
     $$
     上面是DPO的损失，前半段为好答案的相对对数概率，后半段为差答案的相对对数概率，好的部分超出差的部分被sigmoid函数压缩在0-1之间取负对数得到交叉熵

     **Step4**

     **数据准备**:收集 prompt 和对应的生成输出对$(y_1,y_2)$，还有人类或小模型标注偏好$(y_{preferred}>y_{less-preferred})$

     **概率计算**:将每个输出输入大模型，计算生成概率$\pi_{\theta}(y|x)$ , 可以使用 log−probability 累加得到序列概率

     **计算损失**: 使用 DPO 损失函数$L_{DPO}$ , 通过反向传播直接优化大模型参数

     **迭代训练**: 批量计算偏好对损失, 梯度更新模型参数, 不需要价值网络，也不需要 RL 环节

   - DPO的两种变体

     **SimPO：直接去掉参考模型省显存**

     核心思想是不再和“旧模型”比，而是让“好答案的概率”直接大于“差答案的概率”，并且要求好答案概率领先一个**margin**。它做了两处简单修改：一是根据回复长度标准化更新幅度（这个思路后续还会出现），二是移除了参考策略。虽然这破坏了DPO基于策略比值的数学论证，但更纯粹地体现了加权优质/降权劣质的思想。

     原始DPO公式：
     $$
     L_{DPO}(\pi_{\theta}:\pi_{ref})=-E[ln\sigma(\beta ln\frac{\pi_{\theta}(y_w|x)}{\pi_{ref}(y_w|x)}-\beta ln\frac{\pi_{\theta}(y_l|x)}{\pi_{ref}(y_l|x)})]
     $$
     SimPO公式：
     $$
     L_{SimPO}(\pi_{\theta})=-E[ln\sigma(\frac{\beta}{|y_w|} ln{\pi_{\theta}(y_w|x)}-\frac{\beta}{|y_l|}{\pi_{\theta}(y_l|x)}-\gamma)]
     $$
     $\beta$为温度系数，用于控制概率比的敏感度，$\gamma$是超参数，在SimPO中引入一个固定的margin，确保好的回答的概率要比坏的回答的概率至少高$\gamma$

     $|y_w|,|y_l|$分别标识好的回答和坏的回答的长度（token为单位）

     **长度归一化DPO，防止模型“写长文”作弊**

     核心思想是把概率换成**平均每 token 的概率**，再比大小；长文不再天然占优。
     $$
     max_{\pi_{\theta}}E_{y_c,y_r,y_r,D}[ln\sigma(\frac{\beta}{|y_c|} ln\frac{\pi_{\theta}(y_c|x)}{\pi_{ref}(y_c|x)}-\frac{\beta}{|y_r|} ln\frac{\pi_{\theta}(y_r|x)}{\pi_{ref}(y_r|x)})]
     $$
     $|y_c|,|y_r|$是答案的长度（token数）

     在**长度归一化的 DPO** 中，通过将对数概率除以回答的长度，可以**减少模型倾向于生成更长回答的倾向**，因为较长的回答即使在概率上只有轻微的优势，也可能因为长度而获得更高的未归一化的对数概率。
